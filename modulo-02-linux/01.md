# 📘 Aula 01 — Linux para DevOps

> Todo este módulo é executado dentro da VM via VS Code Remote SSH.

---

## 🐧 Por que Linux?

Quando você conecta via SSH em uma EC2, Azure VM ou GCP Instance — você está em Linux. Docker, Kubernetes, CI/CD — tudo roda em Linux. Não tem como separar DevOps de Linux.

---

## 🗂️ Estrutura de diretórios — onde cada coisa fica

```
/
├── etc/      → configurações de serviços (nginx, ssh, docker)
├── var/log/  → logs do sistema e de serviços
├── var/lib/  → dados persistentes (docker, banco de dados)
├── home/     → diretórios dos usuários
├── opt/      → aplicações instaladas manualmente
├── tmp/      → arquivos temporários (limpos no reboot)
└── proc/     → informações do kernel em tempo real
```

**Os que você usa todo dia:**

```bash
cat /etc/hostname               # nome da máquina
cat /etc/hosts                  # mapeamento IP → nome
cat /etc/ssh/sshd_config        # config do SSH
ls /var/log/                    # logs disponíveis
ls /var/lib/docker/             # dados do Docker
```

## ⌨️ Comandos essenciais

**Navegação:**
```bash
pwd                  # onde estou
ls -la               # listar com permissões e dono
cd /etc              # entrar no diretório
cd ~                 # ir para home
cd -                 # voltar ao diretório anterior
```

**Arquivos:**
```bash
touch arquivo.txt               # criar arquivo vazio
echo "texto" > arquivo.txt      # criar com conteúdo
echo "mais" >> arquivo.txt      # adicionar linha
cat arquivo.txt                 # ver conteúdo
tail -f /var/log/syslog         # acompanhar em tempo real
cp arquivo.txt backup.txt       # copiar
mv arquivo.txt /tmp/            # mover
rm -rf /tmp/pasta/              # remover diretório
mkdir -p app/logs               # criar com subpastas
```

**Busca:**
```bash
find /etc -name "*.conf"                    # buscar por nome
grep "error" /var/log/syslog               # buscar dentro de arquivo
grep -r "porta" /etc/                      # buscar recursivo
grep -v "^#" /etc/ssh/sshd_config          # ignorar comentários
ps aux | grep nginx                        # pipe: encadear comandos
```

**Editor vim** (obrigatório em servidores):
```
vim arquivo.txt   → abre
i                 → modo inserção (digitar)
Esc               → modo normal
:wq               → salvar e sair
:q!               → sair sem salvar
/palavra          → buscar
```

## 🔒 Permissões

```bash
ls -la arquivo.txt
# -rw-r--r-- 1 devops devops 220 Jan 10 09:00 arquivo.txt
#  │││└──┴──┴── outros: r--  = só leitura
#  ││└──────── grupo:   r--  = só leitura
#  │└───────── dono:    rw-  = leitura + escrita
#  └────────── tipo:    -    = arquivo
```

**Valores:** `r=4` `w=2` `x=1`

```bash
chmod 755 script.sh        # rwxr-xr-x → scripts
chmod 644 config.yml       # rw-r--r-- → arquivos de config
chmod 600 ~/.ssh/id_ed25519 # rw------- → chaves SSH
chmod +x script.sh         # adicionar execução
chown deploy:deploy /opt/app/  # mudar dono
chmod -R 755 /opt/app/     # aplicar recursivo
```

**Referência rápida:**
```
Scripts:          755
Configs:          644
Chaves SSH:       600
Diretório .ssh/:  700
```

## ⚙️ Serviços com systemctl

```bash
sudo systemctl status nginx       # ver estado
sudo systemctl start nginx        # iniciar agora
sudo systemctl stop nginx         # parar
sudo systemctl restart nginx      # parar + iniciar
sudo systemctl reload nginx       # recarregar config sem parar
sudo systemctl enable --now nginx # habilitar no boot + iniciar
sudo systemctl disable nginx      # remover do boot
```

> `enable` = inicia no boot | `start` = inicia agora  
> Em produção sempre use `enable --now`

**Ver logs do serviço:**
```bash
sudo journalctl -u nginx -f           # tempo real
sudo journalctl -u nginx --since "1 hour ago"
sudo journalctl -u nginx -p err       # só erros
```

## 📋 Logs

```bash
ls /var/log/                          # logs disponíveis
sudo tail -f /var/log/syslog          # sistema em tempo real
sudo tail -f /var/log/auth.log        # autenticações
sudo journalctl -f                    # todos os logs em tempo real
sudo journalctl -u nginx -f           # logs de um serviço
grep "error" /var/log/syslog          # buscar erros
grep -i "failed" /var/log/auth.log    # tentativas de login falhas
```

## 🔧 Shellscript básico

```bash
#!/bin/bash

# Variáveis
APP_NAME="minha-app"
PORT=8080

# Condicional
if systemctl is-active --quiet nginx; then
    echo "nginx está rodando"
else
    echo "nginx parado — iniciando..."
    systemctl start nginx
fi

# Loop
for servico in nginx ssh docker; do
    echo "Status: $servico"
    systemctl is-active $servico
done

# Função
checar_servico() {
    if systemctl is-active --quiet $1; then
        echo "✅ $1 rodando"
    else
        echo "❌ $1 parado"
    fi
}

checar_servico nginx
checar_servico ssh
```

**Executar script:**
```bash
chmod +x script.sh
./script.sh
```

## ⏰ Crontab — agendamento de tarefas

```bash
crontab -e      # editar tarefas agendadas
crontab -l      # listar tarefas agendadas
```

**Formato:**
```
# min  hora  dia  mês  dia-semana  comando
  0    2     *    *    *           /opt/scripts/backup.sh
  */5  *     *    *    *           /opt/scripts/healthcheck.sh
  0    0     *    *    1           /opt/scripts/limpeza.sh
```

```
*  = qualquer valor
*/5 = a cada 5 minutos
0 2 = às 02:00
```
